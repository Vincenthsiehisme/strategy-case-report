### 軸一｜根源追問完整版（The real problem）

**執行方式：** 對第一輪寫出的 real problem 定義，依序執行以下追問。每一層的答案成為下一層的輸入。

```
第一層：「這個問題為什麼存在？」
→ 寫出答案 A

第二層：「答案 A 為什麼沒有被解決？什麼力量讓它持續存在？」
→ 寫出答案 B（結構性阻力）

第三層：「為什麼這個結構性阻力在現在這個時間點變得無法再忽視？」
→ 寫出答案 C（根源 + 時間點）
```

**停止條件：** 答案 C 必須同時滿足以下兩點才能停止追問：
1. 答案 C 和答案 A 的內容明顯不同（如果幾乎一樣，代表沒有真正往下挖）
2. 答案 C 包含一個「結構性力量」——產業趨勢、消費者行為轉變、技術條件、競爭格局變化，其中至少一個

**覆寫規則：** 用答案 C 重寫 real problem 的核心定義句，確保定義句包含根源，而不只是品牌現象描述。

---

### 軸二｜Insight 非顯然性測試完整版（The tension）

**執行方式：** 對第一輪寫出的 insight 句，依序執行以下測試。

```
測試一：「這個 insight 在被我說出來之前，有沒有人說過類似的話？」
→ 在已搜集的資料中找，或依已知產業知識判斷
→ 如果有：記下已有說法，繼續測試二
→ 如果沒有：直接進測試三

測試二：「我的版本和已有說法的差異是什麼？」
→ 寫出差異
→ 如果差異只是換了幾個字或調整了語序：insight 必須重寫，重新執行測試一
→ 如果差異是「命名了一個新的矛盾」：進測試三

測試三：「這個 insight 句，揭示了一個之前沒被明確說出來的內在衝突嗎？」
→ 判斷標準：把這個 insight 句給一個熟悉這個產業的人看，他的反應應該是
   「對，但我之前沒有這樣想過」而不是「這誰都知道」
→ 如果是「這誰都知道」：insight 必須重寫，挖到更深一層的矛盾
→ 如果是「對，但我之前沒有這樣想過」：通過
```

**覆寫規則：** 用通過測試三的版本覆寫 tension 頁的 insight 句和矛盾來源說明。

---

### 軸三｜What to steal 可執行性測試完整版

**執行方式：** 對每個 steal，依序執行以下測試。

```
測試一：「如果使用者明天就要用這個方法，第一步的具體行動是什麼？」
→ 如果答不出具體的第一步（不是「研究市場」這種模糊動作，而是「召集哪些人、做什麼決定、產出什麼」）：
   這個 steal 只是描述對方做的事，必須重寫

測試二：「這個第一步，使用者現在的組織條件能不能做到？」
→ 如果能：把第一步寫入使用條件
→ 如果不能：在使用條件中明確寫出「前置條件：需要先具備 ___」，
   而不是假裝每個人都能直接用
```

**覆寫規則：** 每個 steal 必須有一個可以被執行的第一步，使用條件必須說明前置條件（如果有的話）。

## 重要前提

**批判內化，不外顯。Scorecard 是 AI 的內部執行紀錄，不輸出給使用者確認。**

AI 完成批判後，直接修正分析內容，不停止、不輸出 Scorecard。  
批判執行摘要（發現了什麼問題、如何修正、或為何無法修正）寫入最終報告的**來源附錄**，這是使用者唯一可見的批判紀錄。  
有任何 ❌ 直接修正；無法修正（資料不足）→ 標注「初步判讀，缺乏公開數據支持」，在來源附錄說明。  
批判完成後直接進 Phase 6.5。

## Full Mode｜完整批判問題庫

切換到策略長視角，對第一輪分析進行高壓挑戰：

**問題定義層**
- 這個 problem 是真實問題，還是品牌願望？
- 有沒有把消費者行為的表面觀察，誤認為深層 insight？

**Insight 品質層**
- 這個 insight 是否足夠 specific？具體測試方法：把這個 insight 句子換成另外三個同類品牌的名字，如果都還成立，就是太通用，必須重寫到只有這個案例才能成立為止
- 有沒有只是把現象重新描述一次，然後叫它 insight？例如「消費者越來越重視健康」是觀察，不是 insight；「消費者重視健康，但在結帳的那一刻仍然選擇便宜的選項，因為健康感知沒有被轉化成當下的選擇理由」才是 insight

**論述嚴謹度層**
- 有沒有把活動成功誤寫成品牌成功？
- 哪些地方講太滿？
- 哪些地方只是合理推論，不是被證明的事實？
- 這句話如果被問「你怎麼知道」，扛得住嗎？

**事實 vs. 判讀層**
- 事實與判讀是否清楚區分？
- 有沒有引用不穩定來源卻寫成確定陳述？

## Full Mode｜批判執行邏輯（v2.3 內化版）

完成批判後，AI 直接執行以下動作，不輸出給使用者：
1. **直接修正**：所有 ❌ 項目立即修正分析內容
2. **標注待驗證**：⚠️ 項目在報告正文標注「初步判讀」
3. **來源附錄記錄**：寫入批判執行摘要——發現了什麼問題、如何修正、或為何無法修正
4. **繼續執行**：批判完成後直接進 Phase 6.5，不停止等待確認

## Sprint Mode｜精簡批判（3 個核心問題）

Sprint Mode 中批判壓縮為以下 3 個問題，仍需使用者確認後才進輸出：

1. **這個 insight 如果被換成另外三個品牌來套，成不成立？**  
   → 如果成立，insight 太通用，必須重寫。

2. **有沒有把推論當事實陳述？**  
   → 找出最少 1 個可能過度推論的句子，重寫或加上「初步判讀」標注。

3. **有沒有把活動成效誤寫成品牌成效？**  
   → 若有，明確區分兩者。

Sprint Mode 自查清單（AI 內部執行，不輸出）：

| 問題 | 狀態 |
|------|------|
| Insight 不通用，不能套在三個以上其他案子 | ✅ / ❌ |
| 推論與事實已區分 | ✅ / ❌ |
| 活動成效與品牌成效已區分 | ✅ / ❌ |

全部 ✅ 直接繼續。有 ❌ 立即修正後繼續，不停止等待確認。
