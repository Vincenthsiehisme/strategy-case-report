# 深挖協議｜Deep Dive Protocol

> 此文件由 SKILL.md Phase 5.5 引用。  
> 目標：在第一輪分析完成後，把分析往下再挖一層。  
> 沒有 ✅/❌，只有「挖到這個答案之前不能停」的追問序列，加上收斂機制防止無限下探。  
> 執行結果直接覆寫 Phase 5 對應內容，不輸出給使用者。

---

## 三軸追問總覽

| 軸 | 針對頁面 | 核心問題 | 最大迭代 |
|---|---------|---------|---------|
| 軸一 | The real problem | 為什麼這個問題在現在這個時間點無法再被忽視？ | 3 次 |
| 軸二 | The tension | 這個矛盾存在多久了？為什麼市場習以為常、沒有人解決它？ | 3 次 |
| 軸三 | What to steal | 這個方法在什麼情境下不成立？第一步具體是什麼？ | 2 次 |

---

## 軸一｜根源追問（The real problem）

### 核心邏輯

第一輪分析的 real problem 通常停在「品牌困境描述」層。深挖的目標是找到讓這個困境持續存在的**結構性力量**——一個比品牌本身更大、更難改變的力量，而且要說清楚為什麼它在現在這個時間點變得無法再忽視。

### 追問序列

```
第一層：「這個問題為什麼存在？」
→ 寫出答案 A

第二層：「答案 A 為什麼沒有被解決？是什麼結構性力量讓它持續存在？」
→ 對照下方八類結構性力量清單，找出最吻合的 1–2 類
→ 寫出答案 B

第三層：「為什麼這個結構性力量在現在這個時間點變得無法再忽視？」
→ 找出觸發點：什麼事件、什麼趨勢、什麼臨界點讓它從「可以繼續忍」變成「不能再忍」
→ 寫出答案 C
```

### 八類結構性力量

追問第二層時，從以下清單找根源，不限定只能選一類：

1. **產業趨勢**：技術替代、市場集中化、商業模式位移
2. **消費者行為轉變**：需求結構改變、決策邏輯改變、價值觀位移
3. **技術條件**：新技術出現使舊做法失去優勢、或原本的技術瓶頸被突破
4. **競爭格局變化**：新進入者改寫規則、原有競爭邊界被打破
5. **法規 / 政策約束**：監管收緊、合規成本上升、政策誘因改變
6. **通路 / 供應鏈結構**：通路話語權轉移、中間層被壓縮或取代
7. **組織激勵 / KPI 失配**：內部指標與市場現實脫節，造成系統性誤判
8. **成本結構 / 毛利壓力**：原有商業模式的成本假設已不成立

### 停止條件

答案 C 必須同時滿足：
1. 與答案 A 的內容明顯不同（幾乎一樣代表沒有真正往下挖）
2. 包含至少一個可被指名的結構性力量（從八類中選）
3. 有明確的「時間點觸發邏輯」——說清楚為什麼是現在

### 收斂機制

- 最多追問 3 次。第 3 次仍未滿足停止條件，輸出：「目前材料不足以支撐更深一層的根源定義，保留為待驗證，標注結構性因素方向但不宣稱確定。」
- 不得為了滿足停止條件而虛構結構性因素。若沒有公開材料支撐某類結構性力量，不得宣稱它存在。

## 軸二｜矛盾命名測試（The tension）

### 核心邏輯

第一輪分析的 tension 通常是「觀察的重新描述」。深挖的目標不是找新穎的觀察，而是**命名一個習以為常的矛盾**——一個大家都知道、但沒人質疑、也沒人解決的內在衝突。

真正有力的 insight，是把一個「本來就這樣」的現象說成「其實這件事本身就是一個矛盾」。新穎性是結果，不是判斷標準。

### 追問序列

```
第一層：「這個矛盾存在多久了？」
→ 估算這個矛盾在產業或消費者層面存在的時間跨度
→ 如果存在不到 1–2 年，可能是新現象而非習以為常的矛盾，要重新判斷

第二層：「為什麼市場接受了它、沒有人去解決它？」
→ 找出讓這個矛盾「被習以為常」的原因
→ 可能是：太難改、沒有人有誘因改、大家都假裝不存在、改變的成本高於維持現狀

第三層：「品牌的策略動作，是第一個把這個矛盾說清楚並且採取行動的嗎？」
→ 僅在目前已蒐集的材料範圍內判斷
→ 若無明確證據，不得宣稱「前人沒做過」，只能寫「在目前材料中未見明確同型策略動作」
```

### 雙重測試（第三層完成後執行）

**測試一｜替換測試**
把 insight 句中的品牌名換成 2–3 個同類品牌，看這句話是否仍然成立。若對多數同類品牌都成立，insight 太通用，必須重寫到只有這個案例才能成立。

**測試二｜反駁測試**
問：這個 insight 在什麼情境下不成立？能不能找到一個明確的反例或邊界條件？若找不到任何邊界，insight 可能只是一個普遍真理而非策略判斷，必須重寫到「有邊界」的版本。

好的 insight 不只是「不通用」，還要「有邊界」——說得出它在哪些條件下成立、在哪些條件下不成立。

### 停止條件

通過雙重測試，且第三層的矛盾描述能讓人說出「對，但我之前沒有這樣想過」而不是「這誰都知道」。

### 收斂機制

- 最多重寫 3 次。第 3 次仍未通過雙重測試，輸出：「目前材料不足以支撐更深一層的矛盾定義，保留現有版本並標注『初步判讀，矛盾命名待驗證』。」
- 不得為了通過替換測試而把 insight 寫得過度具體（流於案例描述而非策略判斷）。

## 軸三｜What to steal 可執行性測試

### 核心邏輯

第一輪分析的 What to steal 通常停在「描述對方做的事」。深挖的目標是確保每個 steal 是**真正可借用的方法**，有具體的第一步，且有明確的使用邊界。

### 雙重測試

**測試一｜可執行性測試**
問：「如果使用者明天就要用這個方法，第一步的具體行動是什麼？」

具體的第一步必須能回答：
- 召集哪些人
- 做什麼決定
- 產出什麼

若只能說「研究市場」「評估可行性」這類模糊動作，這個 steal 只是描述對方做的事，必須重寫成可執行的方法。

同時問：「使用者目前的組織條件能不能做到這個第一步？」若不能，在使用條件中明確寫出「前置條件：需要先具備 ___」。

**測試二｜反駁測試**
問：「這個方法在什麼情境下不成立？」

找出至少一個邊界條件——什麼樣的品牌規模、組織結構、市場條件下，這個方法會失效或不適用。沒有邊界的方法是假方法。

### 停止條件

每個 steal 通過雙重測試，有可說出的第一步，且使用條件中已標注邊界。

### 收斂機制

- 最多重寫 2 次。第 2 次仍無法說出具體第一步，將這個 steal 降級為「策略觀察」而非「可借用方法」，並在報告中標注差異。
- 不得為了通過測試而虛構一個看起來具體但實際無法執行的第一步。
