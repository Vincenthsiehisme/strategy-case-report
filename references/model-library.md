## 模型選擇決策樹

**先用決策樹收斂，再主動思考有沒有反直覺配置的可能。**  
決策樹是保底，不是規定。最有價值的配置往往是意外的選擇。

```
Step 1｜案例類型判斷
├── 行為改變型（公益 / 健康 / 政策 / 教育）→ 必含 COM-B
├── Rebranding / 品牌識別重塑型 → 考慮 Kapferer
└── 其他 → 進 Step 2

Step 2｜核心張力判斷
├── 使用情境不明確 / 真實任務與表面需求不一致 → JTBD
├── 品類競爭 / 記憶點 / 品類入口問題 → CEPs
├── 品牌意義需要重建 / 功能價值需要往上翻譯 → Brand Ladder 或 Brand Essence Pyramid
└── 策略信念驅動 / 平台型 / 議題型 → Golden Circle（謹慎使用，見下方規則）

Step 3｜System 頁配置
└── 幾乎永遠用 Consumer Journey

最終每案選出 2–3 個，記錄選擇理由。
```

## 核心模型庫（Core Model Library）

### 1. SWOT → TOWS

**用途**  
先做品牌 / 市場診斷，再把診斷轉成策略方向。

**適合放在**  
- 問題定義頁
- 品牌 / 類別現況頁
- Why now 頁

**注意**  
不要只列清單。一定要從 SWOT 推到 TOWS，否則只是盤點，不是策略。

**建議圖表形式**  
診斷 → 策略動作雙層圖

---

### 2. STP

**用途**  
定義這次真正重要的受眾，以及品牌要搶的位置。

**適合放在**  
- Problem 頁
- Audience 頁
- Positioning 頁

**注意**  
不要只切 demographics。Target 必須有策略意義。Positioning 必須說出品牌想移到哪裡。

**建議圖表形式**  
Segmentation / Targeting / Positioning 三段式圖

---

### 3. JTBD

**用途**  
看受眾在什麼情境下，為了什麼 job 選擇品牌 / 產品。

**適合放在**  
- Tension 頁
- Insight 頁
- Usage / Occasion 頁

**特別適合**  
- 使用情境型品類
- 品牌延伸型案例
- 表面需求與真實任務不一致的案例

**建議圖表形式**  
Context → Job → Tension → Desired Progress

---

### 4. Brand Ladder

**用途**  
把功能利益一路往上推到情緒價值與更高層意義。  
非常適合說明一個產品功能如何被翻譯成更有力的品牌說法。

**適合放在**  
- Strategic Reframe 頁
- Value Translation 頁

**建議圖表形式**  
Functional → Emotional → Meaning 階梯圖

---

### 5. Brand Essence Pyramid

**用途**  
把品牌從 attributes、benefits、values 一路整理到 essence。

**適合放在**  
- Reframe 頁
- Brand Meaning 頁
- 品牌角色頁

**特別適合**  
- 品牌 reframe
- 硬知識品牌（指品牌的核心價值建立在專業知識、技術能力或學術權威上，例如醫療器材品牌、金融服務品牌、工業 B2B 品牌——這類品牌的溝通挑戰在於如何把「我們很專業」翻譯成「你可以信任我們」）
- 品牌角色被重新翻譯的案例

**建議圖表形式**  
Attributes → Benefits → Values → Essence 金字塔圖

---

### 6. Golden Circle

**用途**  
用 Why / How / What 定義品牌為什麼做這件事。

**適合放在**  
- 品牌信念型案例
- 平台型案例
- 議題型案例
- 品牌角色延伸頁

**注意**  
不要濫用。不是每個案例都需要上升到 Why。沒有明確信念驅動時，不要硬用。

**建議圖表形式**  
Why / How / What 同心圓或分層圖

---

### 7. Consumer Journey

**用途**  
把 insight 落到接觸點與 IMC 路徑。System 頁的預設模型。

**適合放在**  
- System 頁（幾乎每案都用）

**建議圖表形式**  
分階段流程圖 + Touchpoint 角色

---

### 8. Category Entry Points（CEPs）

**用途**  
找出品類被想起的情境、觸發點、進入時刻。

**適合放在**  
- Problem 頁
- Tension 頁
- Entry Trigger 頁

**特別適合**  
FMCG、零售、平台品牌、需要搶占品類入口記憶的案例

**建議圖表形式**  
情境 / 觸發點地圖

---

## 條件式模型庫（Conditional Model Library）

以下 4 個模型保留，但不是每案都要用。

### 9. 4P

**用途**  
檢查這次案子是否只停在 Promotion，還是連 Product / Place / Price 都有一起動。

**注意**  
不要把它當 insight 主模型。用於 Execution Check 頁 / 商業落地檢查頁。

---

### 10. Binet & Field

**用途**  
判斷案例偏短期轉換、長期品牌建設，或兩者混合。

**注意**  
當長短期角色分工對案例判讀有價值時才使用。用於 Proof 頁 / Evaluation 頁。

---

### 11. COM-B

**用途**  
專門處理行為改變型案例，拆 Capability / Opportunity / Motivation。

**注意**  
沒有明確行為改變目標時，不要硬用。

---

### 12. Kapferer Brand Identity Prism

**用途**  
用於品牌 identity、rebranding、品牌識別重塑。

**注意**  
只有在需要比 Brand Essence 更完整的 identity 視角時才用。

---

## 模型使用規則

**Rule 1｜每個案例最多使用 3 個模型**  
不要平均分配。不要為了看起來完整而硬塞模型。只選最適合這個案例的 2–3 個。

**Rule 2｜模型是分析鏡頭，不是章節主角**  
模型應該嵌在案例邏輯裡，而不是成為每頁大標。報告不能做成理論教學簡報。

**Rule 3｜模型必須轉成圖表**  
所有被選中的模型，都要做成可上簡報的圖表。模型不能只寫名字。

**Rule 4｜模型的預設放置位置**
- Problem 頁 → 優先：SWOT → TOWS / STP / CEPs / JTBD
- Tension 頁 → 優先：JTBD / CEPs
- Reframe 頁 → 優先：Brand Ladder / Brand Essence Pyramid / Golden Circle
- System 頁 → 優先：Consumer Journey
- Proof / Evaluation 頁 → 視情況補：4P / Binet & Field / COM-B

---

## 圖表設計原則

- 圖表要精緻、可讀
- 圖表不能比主論述更大聲
- 圖表是輔助判斷，不是代替判斷
- 圖表形式參考各模型建議，但可依案例調整

## HTML 圖表實作指引

Phase 7 製作時，所有模型圖表以 HTML/CSS 實作，不用圖片替代。以下為各類圖表的結構方向：

### 階梯型（Brand Ladder、Brand Essence Pyramid）
```html
<!-- 由下往上堆疊，下層最寬，上層最窄 -->
<div class="ladder">
  <div class="ladder-row" style="width:100%">Functional</div>
  <div class="ladder-row" style="width:75%">Emotional</div>
  <div class="ladder-row" style="width:50%">Meaning / Essence</div>
</div>
```
CSS：每層用 `--accent` 色做左側 border 或漸變背景，文字左對齊，層間有明確間距。

### 雙層矩陣型（SWOT → TOWS）
```html
<!-- 上層 SWOT 四象限，下層 TOWS 策略方向 -->
<div class="matrix-grid" style="display:grid; grid-template-columns:1fr 1fr; gap:1px">
  <div class="cell">S</div><div class="cell">W</div>
  <div class="cell">O</div><div class="cell">T</div>
</div>
<div class="tows-row">→ SO策略 / WO策略 / ST策略 / WT策略</div>
```
CSS：格線用背景色差區分，不用 border（視覺更乾淨）。TOWS 行用 `--accent` 色底色。

### 三段式（STP、Golden Circle）
```html
<!-- 橫向三欄，或同心圓（用純 CSS border-radius 實作） -->
<div class="three-col" style="display:grid; grid-template-columns:1fr 1fr 1fr">
  <div>Segmentation</div>
  <div>Targeting</div>
  <div>Positioning</div>
</div>
```
Golden Circle 用三個同心 `div`，外大內小，`border-radius:50%`，每層不同透明度背景。

### 流程型（Consumer Journey、JTBD）
```html
<!-- 橫向流程，節點之間用箭頭符號或 CSS after 偽元素連接 -->
<div class="journey" style="display:flex; align-items:center; gap:0">
  <div class="node">Awareness</div>
  <div class="arrow">→</div>
  <div class="node">Consideration</div>
  <div class="arrow">→</div>
  <div class="node">Action</div>
</div>
```
JTBD 用 `Context → Job → Tension → Progress` 四節點橫排，每節點下方附一行本案的具體內容填入。

### 情境地圖型（CEPs）
```html
<!-- 中心品牌，周圍放射狀排列情境節點 -->
<div class="cep-map" style="position:relative; width:400px; height:400px">
  <div class="center">品牌</div>
  <div class="cep-node" style="position:absolute; top:10%; left:40%">情境A</div>
  <div class="cep-node" style="position:absolute; top:40%; left:80%">情境B</div>
  <!-- 以此類推 -->
</div>
```
CSS：節點之間用 `border` 或 SVG `line` 連接中心，節點用 `--accent` 色圓形底色。

### 通用原則
- 所有圖表顏色使用 CSS 變數（`--bg-secondary`、`--accent`、`--text-primary`），不 hardcode hex 值
- 圖表寬度不超過頁面寬度的 60%，避免圖比文字更重
- 每個圖表下方附一行 Caption 說明這個模型在這個案例中的具體判斷（不是模型名稱，是本案應用的結論）
