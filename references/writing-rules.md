## 寫作規則

### 整體語氣
- 專業，但不要學術：不用「本研究顯示」「由此可見」這類論文用語，改用直接判斷句
- 有判斷，但不要講太滿：見下方「太滿」定義
- 有深度，但不要堆術語：每個英文術語第一次出現時，要有一句中文說明它在這裡的意思
- 能讓策略長快速讀懂，不需要翻譯：每頁的核心判斷必須在前兩句內出現，不能藏在第三段

### 什麼叫「講太滿」——操作定義

以下任何一種情況，都是講太滿：

| 類型 | 範例（錯） | 修正方向 |
|------|-----------|---------|
| 把推論寫成事實 | 「消費者因此對品牌產生信任」 | 改為「初步判讀是⋯⋯，待銷售數據驗證」 |
| 把活動成效等於品牌成效 | 「這次活動讓品牌形象全面提升」 | 改為「活動期間社群聲量上升 X%，品牌長期偏好度尚無公開數據」 |
| 把相關性寫成因果 | 「正因為這個策略，品牌市佔率成長了」 | 改為「市佔率成長發生在策略執行後，但是否為直接因果，尚待驗證」 |
| 過度概括受眾 | 「所有消費者都渴望真實感」 | 收窄為有資料佐證的特定族群 |

**檢查方式**：每個判斷句問一次「你怎麼知道」。能用公開資料回答的，是事實。不能的，標注為判讀。

### 標題規則
- 短句硬切，一頁一個判斷，字數建議 10–20 字以內
- 要有記憶點：讓人讀完能複述這頁說了什麼
- 不要每句都用同一個修辭句型
- **特別注意：避免整份重複「不是…是…」結構**——最多全報告出現 2 次

### 內文結構
建議層次：**判斷 → 解釋 → implication**

- 判斷：這一頁的核心論點是什麼（一句話）
- 解釋：為什麼這個判斷成立（引用事實或邏輯推導）
- Implication：這件事對品牌或策略意味著什麼（往前推一步）

不要寫成研究論文（從背景鋪陳到結論）。  
不要寫成純案例介紹（發生了什麼 → 然後呢 → 結果如何）。  
每段都要問：這句話有沒有往策略意義推進？沒有的句子刪掉。

### 頁尾收束
每頁最後一句是收束句，功能是「比標題再往前推半步」。  
不是重複標題，不是摘要正文，而是留下一個更強的判斷或更尖的問題。  

範例：  
> 標題：「品牌選擇了一個更難走、但更難被複製的路」  
> 頁尾收束：「這個選擇的代價，就是它的護城河。」

## 各頁功能定義與操作說明

| 頁碼 | 頁面名稱 | 核心功能 | 操作定義 |
|------|---------|---------|---------|
| 1 | Case in one line | 一句話定義案例價值 | 格式：「[品牌] 面對 [真實問題]，選擇 [策略動作]，而不是 [原本的做法]。」不先講背景，不先講得了什麼獎 |
| 2 | The real problem | 定義問題 | 說明市場或消費者端的結構性困境——不是「品牌沒聲量」，而是「為什麼照舊做行不通」的根本原因 |
| 3 | The tension | 拆 tension / insight | **Tension** = 消費者或市場中存在的真實矛盾，是兩件看似都對、卻彼此衝突的事。**Insight** = 從這個矛盾中提煉出的判斷，說明品牌的機會在哪。不能停在表面 observation（觀察到的現象），必須到判斷（這意味著什麼） |
| 4 | The strategic reframe | 說明品牌怎麼重做說法 | **Reframe** = 品牌把同一件事，換了一個框架來說，讓它產生新的意義。這頁要說清楚：舊框架是什麼、新框架是什麼、換框架的依據是什麼 |
| 5 | The system | 呈現 IMC 系統 | 不是媒體列表，而是每個接觸點在消費者路徑中扮演的角色。要說明「為什麼這個點要做這件事」，而不只是「這個點做了什麼」 |
| 6 | Why it lands | 檢視被證明的部分 | 明確區分：已有公開數據佐證的事實 vs. 仍是合理推論的判讀。不誇大，不迴避，直接說清楚哪些成效有根據、哪些還是觀察 |
| 7 | What to steal | 萃取可借用的方法 | 每個方法必須附使用條件：「適合用在什麼類型的品牌、什麼情境下」，不是泛用建議，是有條件的方法論 |

## Deck 設計原則

### 整體方向
- 高級時尚感（luxury-fashion-inspired editorial strategy deck）
- 不是花，而是精緻
- 不是炫，而是有審美的秩序

### 必備特質
- 強字體層級
- 留白乾淨
- 模組化版型
- 模型圖表優雅
- 設計感強，但不搶走內容主體

**拼貼式圖片處理（HTML 實作方式）：**
- 局部裁切：`object-fit: cover; object-position: center` 控制顯示區域，搭配固定高度容器
- 圖文交錯：圖片用 `float` 或 CSS Grid `grid-column` 跨欄，文字環繞或並排
- 多圖拼貼：使用 CSS Grid 的不規則分割（例如 `grid-template-columns: 2fr 1fr`），大小圖混排
- 圖片疊加效果：`mix-blend-mode: multiply` 或 `overlay` 讓圖片和背景色融合，增加整體感
- 禁止：不用 `filter: blur` 做背景虛化（視覺廉價感），不用四張等尺寸圖排成 2×2 網格（過於對稱）

### 頁面節奏
整份 deck 不應所有頁面長得一樣。至少要有以下幾種節奏：

| 頁面類型 | 說明 |
|---------|------|
| statement page | 大字、強判斷、留白為主 |
| analysis page | 模型圖表 + 分析文字 |
| system page | IMC 流程圖 + 接觸點圖片 |
| proof page | 數據 / 佐證 + 精簡文字 |
| takeaway page | 可被借走的方法，條列清晰 |

### 配圖執行規則（C 模式）

**核心原則：圖片是證據，不是裝飾。每張圖都必須在分析邏輯中有存在理由。**

#### 執行流程

每頁配圖依以下順序處理：

**Step 1｜判斷這頁需不需要圖**  
對照下方「各頁配圖需求表」，確認這頁屬於哪一類。  
不需要圖的頁面，不要為了視覺平衡硬加圖。

**Step 2｜確認圖的角色**  
在搜圖之前，先寫一句話說明「這張圖要讓讀者更相信這頁論點的什麼」。  
必須是具體的論證角色，不能是「呈現品牌風格」或「增加視覺豐富度」這類非論證理由。  
如果這句話寫不出來，代表不需要這張圖。

**Step 3｜嘗試搜圖**  
用 image_search 工具搜尋。搜尋關鍵字規則見下方。  
搜到的圖必須通過「證據檢查」才能使用（見下方標準）。

**Step 4｜搜不到或不符標準時，輸出配圖需求描述**  
格式如下（每張未能找到的圖都要填）：

```
【配圖需求】
頁面：[頁面名稱]
圖的角色：[這張圖要讓讀者更相信這頁論點的什麼——必須是具體的論證角色，不是「呈現品牌風格」]
建議找圖方向：[品牌官網 / 新聞截圖 / 社群貼文截圖 / 廣告影片截圖 / 其他]
搜尋關鍵字建議：[中文關鍵字] / [英文關鍵字]
來源建議：[官網 / 特定媒體 / YouTube / Instagram / 其他]
替代方案優先順序：
  1. 數據圖表（若有數字可視覺化，優先做成圖表，比找不到的圖更有說服力）
  2. 文字引述（若有官方聲明或媒體直接引句，以引述格式呈現）
  3. 模型圖（若這頁本身有模型配置，以模型圖替代，不要留空）
```

#### 各頁配圖需求表

「選配」的觸發條件：當這頁有一個具體的視覺事實需要被佐證，且文字無法獨立說明時，才配圖。純概念性的判斷頁（例如 tension 頁只有矛盾陳述、沒有具體執行佐證）通常不需要圖。

| 頁面 | 是否需要圖 | 選配觸發條件 | 建議圖片類型 |
|------|-----------|------------|------------|
| Case in one line | 選配 | 品牌視覺辨識度低、讀者對品牌不熟時 | 品牌視覺、產品主視覺（非廣告截圖） |
| The real problem | 通常不需要 | 有具體市場數據或消費者行為報告可視覺化時 | 數據圖表截圖、調查報告圖 |
| The tension | 選配 | 矛盾的兩端有具體視覺對比可呈現時 | 對比圖、情境照（非品牌特定） |
| The strategic reframe | 選配 | 品牌新舊說法的差異有 KV 可直接對比時 | 廣告主視覺、關鍵 KV |
| The system | **必須配圖** | 每個接觸點都需要執行佐證圖；若搜不到，輸出配圖需求描述（這是預期路徑，不是失敗） | 廣告截圖、活動現場、社群貼文、戶外廣告實拍 |
| Why it lands | 選配 | 有具體數據或媒體報導標題可作為視覺佐證時 | 數據圖表、媒體標題截圖 |
| What to steal | 通常不需要 | 方法論有流程圖可強化記憶點時 | 示意圖、簡化流程圖（自製） |

#### 搜尋關鍵字規則

搜圖前，依以下原則組合關鍵字：

- **品牌類圖**：`[品牌英文名] brand visual` / `[品牌名] 廣告`
- **產品類圖**：`[產品名] product` / `[產品名] packaging`
- **情境類圖**：描述情境而非品牌，例如 `convenience store night shopping taiwan`
- **數據圖**：不用搜圖，改用文字引述數據來源，或描述應製作的圖表樣式

**禁止搜尋的類型（工具限制 + 版權風險）：**
- 廣告影片截圖
- 藝人、代言人、真實人物照片
- 活動現場人群照
- 其他品牌的完整廣告 KV

---

#### 證據檢查：搜到的圖能不能用

搜到圖後，用以下三個問題判斷：

1. **這張圖能不能回答「它要證明什麼」這個問題？**  
   不能 → 不用，改輸出配圖需求描述

2. **這張圖是通用視覺，還是和這個案例直接相關？**  
   通用圖庫風格、無法連結到案例本身 → 不用

3. **這張圖有沒有明顯的版權問題？**  
   有疑慮 → 不用，改輸出需求描述，建議使用者從官方來源取得

#### 來源記錄格式

所有成功嵌入的圖，在報告末尾的 Image Source Appendix 中記錄：

```
[頁面名稱] | [圖片描述] | 來源：[網址或出處] | 類型：[搜尋取得 / 官方提供 / 媒體截圖]
```

所有輸出「配圖需求描述」的頁面，在 Image Source Appendix 中標注：

```
[頁面名稱] | [待補] | 需求描述：[簡述需找圖的方向]
```
